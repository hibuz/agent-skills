agent-skills:code-simplification 기술을 호출하세요.

동일한 동작을 유지하면서 최근에 변경된 코드(또는 지정된 범위)를 단순화합니다:

1. CLAUDE.md를 읽고 프로젝트 컨벤션을 학습합니다.
2. 대상 코드를 식별합니다 — 더 넓은 범위가 지정되지 않은 한 최근 변경 사항이 대상입니다.
3. 코드를 수정하기 전에 코드의 목적, 호출자(Callers), Edge case, Test Coverage를 이해합니다.
4. 단순화 기회를 탐색합니다:
   - 깊은 중첩(Deep nesting) → Guard clause 또는 Helper 추출
   - 긴 함수 → 책임에 따라 분리
   - 중첩된 삼항 연산자(Nested ternaries) → if/else 또는 switch
   - 일반적인 이름 → 서술적인 이름
   - 중복된 로직 → 공통 함수
   - 사용되지 않는 코드(Dead code) → 확인 후 제거
5. 각 단순화를 점진적으로 적용합니다 — 변경할 때마다 테스트를 실행하세요.
6. 모든 테스트가 통과하고, 빌드가 성공하며, Diff가 깨끗한지 확인합니다.

단순화 후 테스트가 실패하면 해당 변경을 되돌리고 다시 고려하세요. 결과를 리뷰하기 위해 `code-review-and-quality`를 사용하세요.

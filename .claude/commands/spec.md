agent-skills:spec-driven-development skill을 호출합니다.

사용자가 무엇을 build하려는지 이해하는 것부터 시작합니다. 다음 항목에 대해 clarifying question을 합니다.
1. 목적과 대상 사용자
2. 핵심 기능과 acceptance criteria
3. tech stack 선호와 제약
4. 알려진 boundary(항상 해야 할 일, 먼저 물어봐야 할 일, 절대 하지 말아야 할 일)

그다음 objective, commands, project structure, code style, testing strategy, boundaries의 여섯 핵심 영역을 모두 포함하는 구조화된 spec을 작성합니다.

작성한 spec은 프로젝트 루트의 `SPEC.md`로 저장하고, 다음 단계로 진행하기 전에 사용자 확인을 받습니다.
